Participation Guidelines

How contributors should work on the project

Contribution Workflow

  • Fork the repository before making any changes.
  • Create a separate branch for each feature, enhancement, or bug fix.
  • Write clear, organized, and readable code.
  • Test your changes before submitting them.
  • Use meaningful commit messages.
  • Submit a pull request with a short explanation of your changes.

Developer Expectations

  • Follow the existing design and coding style of the system.
  • Keep documentation updated when adding new features.
  • Make sure your contribution matches the project scope.
  • Communicate with the team before major structural changes.
  • Review feedback professionally and revise when needed.
  • Respect deadlines and assigned development responsibilities.
